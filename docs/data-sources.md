# Data Sources

The screener uses a two-layer data model:

1. `data/mining-universe.ts` keeps the research universe and non-refreshed qualitative fields.
2. `data/official-cache.json` stores low-frequency official/regulatory updates generated by `pnpm refresh:data`.

## Official Filing Data

The refresh script currently pulls free official SEC EDGAR data for SEC-backed tickers:

- SEC ticker mapping: `https://www.sec.gov/files/company_tickers_exchange.json`
- SEC XBRL companyfacts: `https://data.sec.gov/api/xbrl/companyfacts/CIK##########.json`

The script derives only fields that can be mapped from standard XBRL facts, such as revenue, cash, and net debt. If a company uses non-standard tags, IFRS tags not covered by the mapper, or does not expose a comparable value, the cache stores a warning and leaves the existing fallback value untouched.

## Daily Refresh

Run manually:

```bash
pnpm refresh:data
```

Force a refresh even if the cache is less than one day old:

```bash
pnpm refresh:data:force
```

The GitHub Action `.github/workflows/refresh-official-data.yml` runs once daily and commits `data/official-cache.json` only when the cache changes.

## Market Data

Daily market-derived metrics such as market cap, EV/EBITDA, FCF yield, and dividend yield generally require delayed exchange feeds or licensed vendors to be fully reliable across NYSE, TSX/TSXV, ASX, and other venues. The current implementation does not scrape unofficial quote sites. Add a licensed or explicitly permitted provider adapter before replacing those values at scale.
