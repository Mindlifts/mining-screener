# Data Sources

The screener uses a two-layer data model:

1. `data/mining-universe.ts` keeps the research universe and non-refreshed qualitative fields.
2. `data/official-cache.json` stores low-frequency official/regulatory updates generated by `pnpm refresh:data`.
3. `data/market-data-cache.json` and `data/commodity-price-cache.json` store once-daily delayed market data.

## Official Filing Data

The refresh script currently pulls free official SEC EDGAR data for SEC-backed tickers:

- SEC ticker mapping: `https://www.sec.gov/files/company_tickers_exchange.json`
- SEC XBRL companyfacts: `https://data.sec.gov/api/xbrl/companyfacts/CIK##########.json`

The script derives only fields that can be mapped from standard XBRL facts, such as revenue, cash, net debt, shares outstanding, and free cash flow. Free cash flow is calculated from official cash-flow statement facts as operating cash flow minus capital expenditures. If a company uses non-standard tags, IFRS tags not covered by the mapper, or does not expose a comparable value, the cache stores a warning and leaves the existing fallback value untouched.

Source priority:

1. Official filings and regulator APIs: SEC EDGAR, SEDAR+/issuer filings, ASX announcements, annual/interim reports.
2. Exchange or delayed market-data APIs for daily price, market cap inputs, and commodity futures/spot proxies.
3. Company press releases and presentations for non-standard operating metrics such as AISC, production guidance, catalysts, and reserve updates.
4. Secondary sources only as cross-checks, never as the primary source for a stored metric.

## Daily Refresh

Run manually:

```bash
pnpm refresh:data
```

Force a refresh even if the cache is less than one day old:

```bash
pnpm refresh:data:force
```

The GitHub Action `.github/workflows/refresh-official-data.yml` runs once daily and commits the official, market, and commodity caches only when they change.

## Market Data

Daily market-derived metrics such as market cap, EV/EBITDA, FCF yield, dividend yield, and commodity spot prices generally require delayed exchange feeds or licensed vendors to be fully reliable across NYSE, TSX/TSXV, ASX, LME/LBMA, uranium price publishers, coal indexes, and other venues.

The current free adapter uses delayed Yahoo Finance chart data for equity prices and liquid futures proxies. Market cap is recalculated only when the app has both a delayed quote and an official shares-outstanding value. FCF yield is then derived from official free cash flow divided by that market cap.

Commodity tiles read from `data/commodity-price-cache.json` when populated. Gold, silver, copper, and WTI use delayed futures proxies. Uranium and coal are marked stale when the free symbols are not current; those should be replaced with dedicated publisher/vendor feeds before being treated as production-grade.

The implementation does not scrape unofficial quote pages. Add a licensed or explicitly permitted provider adapter before replacing values at scale.
